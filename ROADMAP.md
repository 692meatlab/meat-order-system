# Order Management - AI-Native 기능 로드맵

## Phase 1: 즉시 적용 (v1.1)

> 기존 데이터 구조만으로 바로 구현 가능한 기능

### 1-1. 스마트 자동완성
- **설명**: 거래처명/상품명 입력 시 과거 데이터 기반 자동완성
- **구현**: 기존 vendorMappings/orders 데이터 활용, 프론트엔드 datalist 또는 커스텀 드롭다운
- **영향 파일**: `app.es6.js`, `index.html`
- **상태**: [x] 완료

### 1-2. 이상치 감지
- **설명**: 평소 수량/단가 대비 비정상 주문 자동 하이라이트
- **구현**: 거래처+SKU별 평균/표준편차 계산, 2σ 초과 시 경고 표시
- **영향 파일**: `app.es6.js`, `app.py` (통계 API), `index.html`
- **상태**: [x] 완료

### 1-3. 대시보드 집계
- **설명**: 거래처별/월별/SKU별 자동 통계 차트
- **구현**: orders 데이터 집계 API + Chart.js 또는 순수 CSS 바 차트
- **영향 파일**: `app.py` (집계 API), `app.es6.js`, `index.html`
- **상태**: [x] 완료

## Phase 2: 중기 적용 (v1.2)

> 신규 로직/API 추가가 필요하지만 기존 아키텍처 내에서 구현 가능

### 2-1. 매핑 학습 (미매칭 SKU 자동 제안)
- **설명**: 수동 매핑 이력을 학습하여 미매칭 상품에 대해 유사 SKU 자동 제안
- **구현**: vendor_mappings 이력 기반 유사도 계산 (Levenshtein/포함 매칭)
- **영향 파일**: `app.es6.js` (빠른매칭 모달 확장), `app.py` (유사도 API)
- **상태**: [x] 완료

### 2-2. 엑셀 템플릿 자동 인식
- **설명**: 신규 거래처 엑셀도 컬럼 구조 자동 파악 + 템플릿 자동 저장
- **구현**: vendor_templates 테이블 활용, 매핑 성공 시 자동 저장, 재사용
- **영향 파일**: `app.es6.js` (변환 로직 확장), `app.py` (템플릿 API 활용)
- **상태**: [x] 완료

### 2-3. 배송 상태 자동 추적
- **설명**: 송장번호로 택배사 API 조회 → 배송완료 시 자동 상태 토글
- **구현**: 로젠택배 등 택배사 배송조회 API 연동, 주기적 상태 체크
- **영향 파일**: `app.py` (배송조회 API), `app.es6.js` (자동 갱신 UI)
- **상태**: [x] 완료

### 2-4. 중복 주문 감지
- **설명**: 동일 수령인+주소+상품 조합 경고
- **구현**: 주문 등록 시 기존 orders와 비교, 중복 후보 표시
- **영향 파일**: `app.es6.js` (등록 시 검증), `app.py` (중복 조회 API)
- **상태**: [x] 완료

## Phase 3: 장기 적용 (v2.0)

> 아키텍처 확장 또는 외부 AI 서비스 연동이 필요한 기능

### 3-1. 수요 예측
- **설명**: 과거 주문 패턴(월별/계절별/거래처별)으로 SKU별 수요 예측
- **구현 방향**:
  - 최소 6개월 이상의 주문 데이터 축적 필요
  - 시계열 분석 (이동평균, 지수평활법) 또는 ML 모델
  - 예측 결과를 대시보드에 차트로 시각화
- **필요 기술**: Python pandas/statsmodels, Chart.js
- **선행 조건**: Phase 1-3 대시보드 집계 완료, 충분한 데이터 축적
- **상태**: [ ] 미착수

### 3-2. 자연어 주문 조회
- **설명**: "지난달 A거래처 미입금 주문" 같은 자연어로 검색
- **구현 방향**:
  - Claude API 연동하여 자연어 → SQL 쿼리 변환
  - 또는 키워드 기반 규칙 엔진으로 경량 구현
  - 검색 결과를 기존 주문관리 테이블 형태로 표시
- **필요 기술**: Claude API (또는 규칙 기반 파서)
- **선행 조건**: 주문 데이터 충분히 축적, API 비용 검토
- **상태**: [ ] 미착수

### 3-3. 자동 발주서 생성
- **설명**: 재고 부족 감지 시 자동으로 발주서 초안 생성
- **구현 방향**:
  - SKU별 적정 재고 수준 설정 기능 추가
  - 출고량 대비 재고 부족 시 알림 + 발주서 초안 자동 작성
  - 거래처별 단가/납기 이력 기반 최적 발주처 추천
- **필요 기술**: 재고 관리 테이블 추가, 알림 시스템
- **선행 조건**: Phase 3-1 수요 예측 완료, 재고 관리 모듈 구축
- **상태**: [ ] 미착수

### 3-4. 모바일 알림 시스템
- **설명**: 배송완료/입금확인/이상치 감지 시 모바일 푸시 알림
- **구현 방향**:
  - Telegram Bot 또는 카카오톡 알림톡 연동
  - 이벤트 기반 알림 트리거 (상태 변경 시)
- **필요 기술**: Telegram Bot API 또는 카카오 비즈메시지
- **선행 조건**: Phase 2-3 배송 자동 추적 완료
- **상태**: [ ] 미착수
