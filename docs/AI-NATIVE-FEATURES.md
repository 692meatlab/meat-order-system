## A-1: 퍼지 SKU 매칭

### 무엇을 하나요?
발주서에서 상품명이 정확히 일치하지 않아도 자동으로 SKU에 매칭합니다.
예: "한우 등심세트 1KG" → "한우등심세트 1kg" (92% 일치)

### 4단계 매칭 파이프라인
1. **정확 매칭**: vendor_mappings에서 거래처+상품명 완전 일치
2. **별칭 매칭**: 사용자가 등록한 대체 상품명
3. **토큰 매칭**: 텍스트를 토큰으로 분리 후 Jaccard 유사도 ≥ 0.6
4. **Trigram 매칭**: PostgreSQL pg_trgm 기반 유사도 ≥ 0.3

### 사용법
- 발주서 변환 시 미매칭 항목에 자동 적용
- 매칭 결과에 confidence 표시: ≥0.8 초록, ≥0.6 노랑, <0.6 빨강
- 별칭 관리: 거래처별 상품관리에서 등록

### API
| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | `/api/fuzzy-match` | 상품명 배열 → 매칭 결과 |
| GET | `/api/matching-aliases` | 별칭 목록 |
| POST | `/api/matching-aliases` | 별칭 등록 |
| DELETE | `/api/matching-aliases/:id` | 별칭 삭제 |

**요청 예시:**
```json
POST /api/fuzzy-match
{
  "items": ["한우 등심세트 1KG", "돼지갈비 세트"],
  "vendor_name": "거래처A"
}
```

**응답:**
```json
{
  "results": [
    {
      "original": "한우 등심세트 1KG",
      "matched_sku": "한우등심세트 1kg",
      "sku_id": 5,
      "confidence": 0.92,
      "match_stage": "token",
      "candidates": [...]
    }
  ]
}
```

## A-2: 원가 이상 감지

### 무엇을 하나요?
부위별/포장재 원가 수정 시 통계적으로 비정상적인 가격 변동을 자동 감지하여 경고합니다.

### 감지 알고리즘
- **이력 3건 미만**: 직전 대비 30% 이상 변동 시 경고
- **이력 3건 이상**: Z-score 기반
  - Z-score > 3: 위험 (빨강) - 극단적 이상치
  - Z-score > 2: 주의 (노랑) - 주의 필요
  - 그 외: 정상

### 사용법
- 원가관리에서 부위/포장재 가격 수정 시 자동 감지
- 이상 감지 시 경고 모달 표시
- 대시보드 알림에 미확인 이상 건수 표시

### API
| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/cost-anomalies` | 미확인 이상 감지 목록 |
| POST | `/api/cost-anomalies/:id/acknowledge` | 확인 처리 |

기존 PUT `/api/parts-cost`, PUT `/api/packaging-cost` 응답에 `anomaly` 필드 추가.

## A-3: 수익성 분석

### 무엇을 하나요?
SKU별 원가를 자동 계산(구성품 × 부위단가 + 포장재)하고, 판매가 대비 마진율을 실시간 표시합니다.

### 마진 등급
| 등급 | 마진율 | 의미 |
|------|--------|------|
| A | > 40% | 고수익 상품 |
| B | 25~40% | 양호 |
| C | 10~25% | 개선 필요 |
| D | < 10% | 긴급 점검 |

### 사용법
- 사이드바 "📊 수익성 분석" 클릭
- SKU별 원가/마진율/등급 테이블 확인
- 클릭하면 구성품별 비용 분해 상세 모달
- 개선 제안 자동 생성 (원가 비중 50% 초과 부위, 포장재 15% 초과 등)

### API
| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/profitability` | 전체 SKU 수익성 목록 |
| GET | `/api/profitability/:sku_id` | 개별 SKU 원가 상세 |
| GET | `/api/profitability/trends?months=6` | 월별 마진 추이 |

## A-4: 거래처 성과 대시보드

### 무엇을 하나요?
거래처별 배송/입금/세금계산서 처리율을 종합 점수화하여 우수/주의 거래처를 즉시 파악합니다.

### 성과 점수 공식
```
총점 = 배송완료율(30%) + 입금완료율(25%) + 세금계산서율(15%)
     + 처리속도점수(20%) + 주문빈도점수(10%)
```

### 등급
| 등급 | 점수 | 색상 |
|------|------|------|
| S | ≥ 90 | 보라 |
| A | ≥ 75 | 파랑 |
| B | ≥ 60 | 초록 |
| C | ≥ 40 | 노랑 |
| D | < 40 | 빨강 |

### 사용법
- 매출 리포트 페이지에서 "성과" 탭 클릭
- 기간 선택: 30일 / 60일 / 90일
- 카드 클릭 시 상세 모달 (월별 추이)

### API
| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/vendor-performance?period=30` | 거래처별 성과 |
| GET | `/api/vendor-performance/:vendor/detail?period=90` | 거래처 상세 |

## A-5: 스마트 중복 감지

### 무엇을 하나요?
주문 등록 시 수령인+전화+주소+SKU를 복합 유사도로 비교하여 중복 주문을 사전 감지합니다.

### 복합 유사도 점수
```
총점 = 수령인(35%) + 전화번호(25%) + 주소(25%) + SKU(15%)
```

| 점수 | 의미 | 표시 |
|------|------|------|
| ≥ 80 | 높은 중복 가능성 | 빨강 경고 |
| ≥ 60 | 중간 | 노랑 주의 |
| < 60 | 낮음 | 무시 |

### 정규화
- 수령인: 공백/직함(님,사장,대표 등) 제거
- 전화번호: 숫자만 추출
- 주소: 특별시→시, 공백 제거

### 사용법
- 주문 등록 전 자동 중복 감지 실행
- 중복 발견 시 비교 모달 (새 주문 vs 기존 주문)
- "중복 아님" 선택하면 제외 목록에 등록

### API
| 메서드 | 경로 | 설명 |
|--------|------|------|
| POST | `/api/orders/check-duplicates` | 중복 감지 (강화) |
| POST | `/api/duplicate-exclusions` | 중복 제외 등록 |

## A-6: 수요 예측

### 무엇을 하나요?
과거 주문 데이터를 기반으로 향후 7/14/30일 SKU별 예상 주문량을 산출합니다.

### 예측 알고리즘
1. **가중 이동 평균(WMA)**: 최근 14일 데이터, 최근일수록 높은 가중치
2. **요일 보정**: 요일별 주문 비율 반영 (월~일)
3. **시즌 보정**: 추석(9월), 설날(1~2월), 연말(12월) → ×1.3~1.5 부스트

### 사용법
- 사이드바 "📈 수요 예측" 클릭
- 기간 선택: 7일 / 14일 / 30일
- SKU별 예측 테이블: 예측량, 신뢰구간, 추세(↑↓→)
- 부위별 소요량 섹션: 구성품 연동하여 필요 부위 kg 산출

### API
| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/forecast?days=7` | SKU별 예측 결과 |
| GET | `/api/forecast/parts?days=7` | 부위별 소요량 예측 |
| GET | `/api/forecast/accuracy` | 예측 정확도 (MAPE) |

## A-7: 스마트 발주

### 무엇을 하나요?
수요 예측(A-6) 결과와 현재 재고를 비교하여 발주 필요량을 자동 산출하고, 거래처별 묶음 발주서를 생성합니다.

### 발주 추천 로직
```
필요량 = max(0, 예측수요 - 현재재고 + 안전재고)
```

### 긴급도
| 등급 | 조건 | 색상 |
|------|------|------|
| critical | 현재고 < 최소재고 또는 일평균 = 0 | 빨강 |
| high | 재고 소진 3일 이내 | 주황 |
| medium | 재고 소진 7일 이내 | 노랑 |
| low | 7일 이상 여유 | 초록 |

### 사용법
- 사이드바 "🛒 스마트 발주" 클릭
- 추천 테이블에서 체크박스로 항목 선택
- 수량 인라인 수정 가능
- "발주서 생성" 클릭 → 거래처별 엑셀 다운로드

### API
| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/smart-order/recommendations?days=7` | 발주 추천 목록 |
| POST | `/api/smart-order/generate` | 발주서 데이터 생성 |
